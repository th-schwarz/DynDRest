ApiController

Up 
get /info/update-log
Delivers pageable update logs. (deliverLogs)

Query parameters

page (optional)
Query Parameter — Page number to fetch. If it is not committed, it will be '1'.
search (optional)
Query Parameter — String to search for. It belongs to 'host' and 'timestamp'.

Return type

UpdateLogPage

Example data

Content-Type: application/json
{
  "total" : 0,
  "totalPage" : 6,
  "pageSize" : 5,
  "page" : 1,
  "items" : [ {
    "dateTime" : "dateTime",
    "ipv4" : "ipv4",
    "ipv6" : "ipv6",
    "host" : "host"
  }, {
    "dateTime" : "dateTime",
    "ipv4" : "ipv4",
    "ipv6" : "ipv6",
    "host" : "host"
  } ]
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Ordered list of update logs. UpdateLogPage
Up 
get /info/{host}
Determines the IP settings of the 'host' and returns it in a JSON object. (info)

Path parameters

host (required)
Path Parameter — The host, for which the IPs must be determined.

Query parameters

apitoken (required)
Query Parameter — The 'apitoken' which belongs to the 'host'.

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

404

If the 'host' isn't configured.

500

If the zone info fails.

200

A JSON object with the IP settings of the 'host'.

Example data

Content-Type: application/json
{"ipv4":"127.1.2.4","ipv6":"2a03:4000:41:32:0:0:0:2"}

400

If the 'apitoken' doesn't belong to the host, IP addresses aren't valid or the remote IP couldn't determine.
Up 
get /update/{host}
Updates the desired IP addresses of the 'host', if the 'apitoken' belongs to the host. If both parameters for IP addresses aren't set, an attempt is made to fetch the remote IP. (update)

Path parameters

host (required)
Path Parameter — The host, for which the IPs must be updated.

Query parameters

apitoken (required)
Query Parameter — The 'apitoken' which belongs to the 'host'.
ipv4 (optional)
Query Parameter —
ipv6 (optional)
Query Parameter —

Responses

200

Updates the desired IP addresses of the 'host', if the 'apitoken' belongs to the host. If both parameters for IP addresses aren't set, an attempt is made to fetch the remote IP.

400

If the 'apitoken' doesn't belong to the host, IP addresses aren't valid or the remote IP couldn't determine.

500

If the update fails.
